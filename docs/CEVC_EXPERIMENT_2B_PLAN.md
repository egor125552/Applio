# CEVC Experiment 2B — исправление Roughness Adapter без новых записей

Статус: утверждённый следующий этап после первого реального A/B-теста.

## Главное решение

Новые записи пользователя для Experiment 2B **не требуются**.

Текущий набор данных замораживается и используется повторно:

- 3 исходные записи: clean, rough и mixed;
- суммарно 7 минут 20 секунд;
- 170 подготовленных срезов;
- 48 clean, 61 rough и 61 mixed срез;
- отдельная запись `Новая запись 278.mp3` используется как фиксированный внешний тест, но не добавляется в GitHub и остаётся на Google Drive.

Пользователю не нужно заново читать одинаковые фразы. Парное обучение будет строиться программно из уже имеющихся записей и синтетических контролируемых преобразований.

## Результат первого прототипа

Первый Roughness Adapter доказал только инженерную часть:

- отдельный checkpoint обучается и загружается;
- базовая RVC-модель остаётся замороженной;
- `roughness = 0` обходит адаптер;
- адаптер влияет на итоговый звук.

Акустический gate не пройден.

Проверенные результаты первого A/B:

| Вариант | Длительность | RMS |
|---|---:|---:|
| Исходник | 27.336 с | -15.71 dB |
| Roughness 0.0 | 27.135 с | -15.62 dB |
| Roughness 0.5 | 27.320 с | -16.13 dB |
| Roughness 1.0 | 26.623 с | -16.26 dB |

У `roughness = 1.0` последний отсчёт равен примерно `-0.1806`, то есть файл заканчивается в середине активной волны. Это подтверждает реальный обрыв, а не только субъективное впечатление.

Слуховой и технический вывод:

- `0.5` и `1.0` различаются недостаточно;
- увеличение roughness в основном делает звук тише/мутнее;
- естественный управляемый хрип не сформирован;
- повторное обучение текущей схемы на 200 эпох или немедленное увеличение до миллиона параметров нецелесообразно.

## Почему первая схема не сработала

Текущий trainer обучает адаптер в режиме реконструкции целевой записи через posterior latent. Основной loss состоит из mel L1 и waveform L1.

При реальной конвертации latent строится другим путём — из content features через prior и reverse flow. Поэтому обучение и inference видят разные распределения latent.

Кроме того, loss не требует от модели:

- монотонного роста хрипотцы при увеличении control;
- сохранения громкости;
- сохранения разборчивости;
- различия между `0.5` и `1.0`;
- совпадения длительности всех A/B-версий;
- отдельного управления HNR и апериодичностью.

## Этап 1 — починить измерительный стенд

До нового обучения исправляется A/B-путь.

- [ ] Один раз извлекать ContentVec, F0, index features и latent.
- [ ] Из одного и того же latent декодировать `0.0`, `0.5` и `1.0`.
- [ ] Не запускать весь RVC pipeline три независимых раза.
- [ ] Гарантировать одинаковое количество кадров и отсчётов у трёх результатов.
- [ ] При несовпадении длины считать тест проваленным, а не молча сохранять WAV.
- [ ] Добавить защитный короткий fade только после исправления длины; fade не должен скрывать обрезание.
- [ ] Автоматически сохранять JSON-отчёт: duration, RMS, peak, clipping, endpoint jump, HNR, spectral flatness и band energy.
- [ ] Добавить regression test на фиксированном synthetic input и проверку одинаковой длины.

## Этап 2 — повторно использовать существующие данные

Новые фразы не записываются.

### 2.1. Честное разделение

Срезы нельзя случайно перемешать между train и validation, потому что соседние срезы одной длинной записи почти одинаковы.

- [ ] Делить данные непрерывными временными блоками.
- [ ] Оставить отдельные блоки clean, rough и mixed только для validation.
- [ ] Не использовать внешний `Новая запись 278.mp3` в обучении.

### 2.2. Программные псевдопары

Из существующей clean-речи создаются контролируемые варианты той же самой фразы:

- слабая и сильная band-limited aperiodicity;
- небольшой jitter;
- небольшой shimmer;
- управляемый spectral tilt;
- лёгкие субгармоники только на voiced-участках;
- несколько комбинаций этих эффектов.

Такая синтетика не считается финальной целью качества. Она нужна, чтобы модель впервые увидела одну и ту же фонетическую фразу при известных значениях roughness.

Настоящие rough и mixed записи остаются главным образцом естественного распределения хрипотцы.

## Этап 3 — отдельный Roughness Critic

На существующих 170 срезах обучается маленький анализатор, который оценивает выраженность roughness независимо от текста.

Входы:

- log-mel/STFT;
- HNR proxy;
- band aperiodicity;
- F0 instability;
- spectral tilt;
- energy envelope.

Выход:

- скаляр roughness `0..1`;
- дополнительная классификация clean / mixed / rough.

Critic после обучения замораживается. Он нужен как измеритель направления, чтобы адаптер не мог снизить loss простым затемнением или уменьшением громкости.

## Этап 4 — Roughness Adapter v2

Сначала сохраняется небольшой размер. Немедленный переход к миллиону параметров запрещён до прохождения акустического gate.

Порядок ёмкости:

1. текущие 60 тысяч параметров с новым loss;
2. при недостаточной мощности — примерно 200–300 тысяч;
3. только после подтверждения правильного направления — конфигурация около 1.03 млн параметров (`hidden_channels=256`, 7 temporal blocks).

Новый training step создаёт несколько выходов из одного latent:

- `r = 0.0`;
- случайное `r_low`;
- случайное `r_high`, где `r_high > r_low`;
- целевое значение, связанное с clean/mixed/rough или синтетической псевдопарой.

Loss добавляется по частям:

- identity/baseline consistency для `r = 0`;
- multi-resolution STFT;
- content consistency через замороженный ContentVec;
- F0/voicing consistency;
- loudness-envelope consistency;
- roughness target loss через замороженный critic;
- monotonic ranking: critic(`r_high`) должен быть выше critic(`r_low`);
- HNR и band-aperiodicity loss;
- штраф за изменение общей громкости и чрезмерное затемнение;
- residual regularization, чтобы адаптер не разрушал latent.

## Этап 5 — критерии успеха

Experiment 2B проходит gate только при одновременном выполнении условий:

1. `0.0`, `0.5` и `1.0` имеют одинаковую длину.
2. Конец WAV находится около нуля и не обрывается на активной волне.
3. `0.0` совпадает с baseline в пределах численной погрешности.
4. `0.5` слышимо отличается от `0.0`, а `1.0` — от `0.5`.
5. Roughness-метрика растёт монотонно.
6. RMS не падает более чем примерно на 0.5 dB без отдельной причины.
7. Content/F0 сохраняются.
8. Нет явной металлической фазы, щелчков и потери согласных.
9. Улучшение слышно минимум на validation-блоках и на внешнем фиксированном MP3.

Если новый loss задаёт правильное направление, но эффект слабый, увеличивается размер адаптера. Если направление остаётся неправильным, переход выполняется к Dual-Source Exciter, а не к большему числу эпох.

## Что требуется от пользователя

На текущем этапе — ничего записывать не нужно.

После реализации Experiment 2B пользователю понадобится только прослушать фиксированный набор A/B-файлов и сказать, какой вариант звучит естественнее. Все подготовка данных, псевдопары, обучение, метрики и проверки выполняются программно.
